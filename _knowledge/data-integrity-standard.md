# Data Integrity — Non-Negotiable Platform Standard

Koto Platform 11 never uses hardcoded, recalled, or AI-generated data as
ground truth for any real-world fact. This rule applies to ALL data
lookups: geography, industry, business listings, reviews, rankings,
categories, demographics, citations.

## The rule, stated simply

If the data can change in the real world, it must be fetched from an
authoritative source. If it was fetched, it must carry a timestamp and a
source URL. If it was generated by AI, it must be flagged as AI-generated
in the UI so the user knows not to trust it without verification.

## Three layers every data lookup must pass through

```
1. REQUEST LAYER  — what the user asks for
2. SOURCE LAYER   — fetch from an authoritative API (never recall / hardcode)
3. STORAGE LAYER  — wrap in VerifiedDataSource with source_url + fetched_at
```

If any layer is skipped, the data must not be shown to users.

## Authoritative sources — the only places Koto fetches real-world data

- **US Geography**: `api.census.gov` — states, counties, cities, ZCTAs
- **Industry codes**: Census NAICS (XLSX, see TODO in `dataSources.ts`), OSHA SIC
- **GBP categories**: Google My Business Information API (live, not cached lists)
- **Business data**: Google Maps Places API, Google Business Profile API
- **Citation directories**: Moz Local or BrightLocal API (not hardcoded lists)
- **Reviews**: platform APIs (Google, Yelp, Facebook) — never cached counts
  without a timestamp

All of the above are registered in `src/lib/dataSources.ts`. If a new data
type is needed, add it to that registry **first** before writing any fetch
logic.

## What is acceptable to hardcode

- **`STATE_FIPS`** in `src/lib/geoLookup.ts` — permanent federal standard
  dating to 1874, codified in ANSI INCITS 38-2009. Unchanged and unchangeable.
- **UI strings, labels, copy** — not data, presentation.
- **Configuration constants** — API endpoints, timeout values, pricing tiers.
- **Small LLM prompt hints** — e.g. the `SIC_HINT` string in
  `src/lib/scoutQueryParser.ts` gives Claude a seed of common industry → SIC
  mappings. Acceptable because it is a hint, not a source of truth — the LLM
  can return any SIC. Must be documented as a hint with a comment.
- **Demo / test data generators** — e.g. `syntheticCallGenerator.ts`.
- **Nothing else geographic, demographic, or industry-related.**

## Key files

- `src/lib/dataIntegrity.ts` — types, stale thresholds, `createVerifiedData`
- `src/lib/dataSources.ts` — registry of every authoritative source URL
- `src/lib/geoLookup.ts` — Census API wrappers: states, counties, places, zips
- `src/lib/geoCache.ts` — server-side cache with expiry (Map-backed; swap for
  KV in production if cold-start cache loss becomes a concern)
- `src/components/DataSourceBadge.jsx` — UI provenance badge (fresh / aging /
  stale grading, optional AI-generated warning, optional cross-reference tick)
- `_knowledge/data-integrity-audit.md` — the audit that established the
  baseline compliance state of the codebase (2026-04-11)

## When in doubt — ask

"Where does this data actually come from?"

If the answer is "I know it" or "it's in the code already" — that is wrong.
The answer must be a URL to a live API or an official source, and the data
must be wrapped in a `VerifiedDataSource` record before it leaves the server.

## Stale thresholds (see `dataIntegrity.ts` for full list)

- **Geographic**: 6-12 months (boundaries change rarely)
- **Industry classifications**: 1 year (NAICS revises every 5 yrs)
- **GBP categories**: 90 days (Google updates these)
- **Business listings / contacts**: 30 days
- **Citation directories**: 90 days
- **Reviews**: 7 days
- **Rankings / live GBP**: 24 hours

The `DataSourceBadge` component grades freshness automatically:
- `fresh` (green) — under 70% of the threshold
- `aging` (amber) — 70-100% — verify soon
- `stale` (red) — over 100% — must refetch before showing

## Adding a new data source

1. Add the source to `DATA_SOURCES` in `src/lib/dataSources.ts` with its
   live URL, docs link, and `threshold_key`.
2. If it needs runtime substitution, document the `{PLACEHOLDER}` tokens in
   the URL and use `buildSourceUrl(key, params)`.
3. Write the fetcher in the appropriate lib file (`geoLookup.ts`,
   `industrySources.ts`, etc.) and wrap the result with `createVerifiedData`.
4. If the source returns bulk data, add a cache key in `cacheKeys` in
   `src/lib/geoCache.ts` and wrap the fetcher with `getOrFetch`.
5. If the source is broken or not yet implemented, mark its registry entry
   with `broken: true` and a note explaining what blocks it.
