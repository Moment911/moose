# Phase 8 — UI Design Contract

> Visual and interaction contract for the Phase 8 external source parsers. Four new ingest modes (form links, website crawls, GBP connections, file uploads) plug into the Phase 7 LaunchPage canvas without replacing it; a new Agency Settings "Integrations" tab holds provider keys, OAuth tokens, and per-day budgets.
>
> Generated by `gsd-ui-researcher`. Consumed by planner, executor, and ui-checker. This spec **inherits all tokens, typography, color, motion, and accessibility rules from the Phase 7 UI-SPEC** (`07-UI-SPEC.md`) and only specifies what is new or different in Phase 8. Where Phase 8 is silent, Phase 7 governs. No new color tokens. No new font stacks. No new motion primitives. Shell chrome (sidebar + top nav + KotoIQ shell tabs) is unchanged.

---

## 0. Phase Overview (executor orientation)

- **Primary surface (extended):** `src/views/kotoiq/LaunchPage.jsx` — unchanged page root; new behavior reached via an extended `IngestPanel` and a page-wide drop zone that now executes instead of deferring.
- **Component family (extended/new):** `src/components/kotoiq/launch/*` gains per-source options panels and a GBP wizard; `src/views/AgencySettingsPage.jsx` gains a new `integrations` section.
- **Four new ingest entry points on the LaunchPage:**
  1. **Form URL** (Typeform / Jotform / Google Forms / generic HTML) — URL pattern detection → form-parser options panel.
  2. **Website URL** — URL pattern detection → crawl options panel (scope / JS rendering / robots.txt / cost cap).
  3. **Connect GBP** — dedicated button opens a 3-mode connection wizard (Agency OAuth / Per-client OAuth / Public Places API).
  4. **File upload** — drop zone (already present from Phase 7) now executes uploads for PDF/DOCX/PNG/JPEG/WebP/HEIC, with a per-file options panel before extraction fires.
- **North-star UX beat (extends Phase 7's "wow"):** "Drop whatever you have, we'll figure it out." Operators see the machine making cost-aware, permission-aware, source-typed decisions in plain sight — with per-instance override of every default.
- **Anti-patterns explicitly rejected (in addition to Phase 7's list):** black-box "just go" flows with no cost preview; modal stacks; auto-expanded options panels that push the "Go" button offscreen; public-data-only paths that look identical to authenticated paths.
- **Three visual contracts unique to Phase 8:**
  - The **per-instance options panel** pattern (disclosure-collapsed, cost-preview-always-visible, single "Go" trigger) — see §5.2.
  - **Source-authenticity chrome** — authenticated sources look different from public/fallback sources so operators never mistake one for the other — see §5.5.
  - **Cost guardrail affordances** — daily-budget ring + override-with-justification flow — see §5.7.

## 3. Color

Unchanged palette. What IS new: a prescriptive mapping of sources to Phase 7's provenance-chip color system (§10 of Phase 7 UI-SPEC), plus an explicit **authenticity tier** visual so public/fallback sources are visually distinguishable from authenticated sources.

### 3.1 Provenance chip colors (extends Phase 7 §10 table)

| `source_type` value | Chip icon | Chip label | Chip color | Authenticity tier |
|-------------------|----------|-----------|-----------|-------------------|
| `typeform_api` | `FileInput` | `typeform` | `T` (teal) | **authenticated** |
| `jotform_api` | `FileInput` | `jotform` | `T` | **authenticated** |
| `google_forms_api` | `FileInput` | `google form` | `T` | **authenticated** |
| `form_scrape` | `Globe` | `form (scrape)` | `AMB` (amber) | **scraped** |
| `website_scrape` | `Globe` | `website` | `AMB` | **scraped** |
| `gbp_authenticated` | `MapPin` | `gbp` | `T` | **authenticated** |
| `gbp_public` | `MapPin` | `gbp (public)` | `AMB` | **public** |
| `pdf_text_extract` | `FileText` | `pdf` | `T` | **uploaded (text)** |
| `pdf_image_extract` | `FileText` | `pdf (ocr)` | `AMB` | **uploaded (ocr)** |
| `docx_text_extract` | `FileText` | `docx` | `T` | **uploaded (text)** |
| `image_ocr_vision` | `Image` | `image` | `AMB` | **uploaded (ocr)** |

**Rule:** The chip color is prescriptive — teal `T` for sources that return structured ground truth (provider APIs, authenticated GBP, text-extracted docs); amber `AMB` for sources that rely on heuristic extraction (scrapes, OCR, public-data fallbacks). This visual tier tells the operator at a glance "how much can I trust this?" without reading the chip label.

### 3.2 Authenticity-tier overlay on GBP connection

Mode 3 (Public Places API fallback) is the single highest-risk UX collision in Phase 8 because the data looks identical in shape to Mode 1/2 but is coarser and never private. It MUST be visually distinct wherever it appears:

- GBP wizard Mode 3 card: 2px dashed `AMB` top border + `AMB + '0d'` background tint (5% amber). Modes 1 and 2: solid `T` 1px border, white background.
- In the briefing doc after a GBP Mode 3 pull, the `gbp (public)` chip is AMB. Hover tooltip includes: `Public data only — no review themes, no private fields.`
- In the Integrations tab, an active Mode 3 connection shows an AMB status dot, not a GRN one. Copy: `Public fallback — no OAuth token stored.`

### 3.3 Daily budget ring color thresholds (D-22 / D-23)

| Utilization | Ring color | Context |
|-------------|-----------|---------|
| 0 – 70% | `GRN` | Nominal |
| 70 – 80% | `GRN` with faint pulse | Heads-up — nothing blocks yet |
| 80 – 100% | `AMB` | Warn (D-22/D-23 trigger), override still automatic |
| 100%+ | `R` (pink) | Block — override required before any new triggered action |

**Rule:** Phase 8 does NOT introduce a destructive red (`DST = '#DC2626'`). Over-budget block uses Koto Pink `R` because it is gated, not destructive — destructive red is reserved for field rejection (Phase 7 §3).

### 3.4 Accent reserved list (additive to Phase 7 §3)

Phase 7 already reserves:
- **Pink (`R`)** — discrepancy catcher, Launch button at ≥90% confidence, high-severity hotspot dots, Reject-field destructive confirmation.
- **Teal (`T`)** — confident halos, citation chips, chat orb, streaming cursor, margin-note left border, saved pulse.

Phase 8 adds:
- **Pink (`R`)** — ALSO used for: the "budget exceeded" ring state, the "Override this cap" button when a cost-cap block fires, the "Blocked" label on an over-limit operation. (Same semantic family as discrepancy: "attention, something wants your decision.")
- **Teal (`T`)** — ALSO used for: the "Go" button on per-instance options panels (matches Phase 7 ingest `Pull it in` CTA — same teal primary), the GBP Mode 1/2 recommended-pick highlight, the upload-complete pulse.
- **Amber (`AMB`)** — ALSO used for: scraped-source chips (§3.1), GBP Mode 3 dashed border (§3.2), 80-99% budget ring, robots.txt warn indicator on website-crawl options panel.

Phase 8 introduces **no new accent slots**. Everything above reuses tokens the shell already owns.

### 4.1 IngestPanel — URL detection branching

When an operator pastes a URL into the Phase 7 `IngestPanel` URL tab, the panel detects the URL pattern BEFORE triggering ingest. The detection result drives which options panel expands.

| Detected pattern | Banner copy (above input, 12px, muted) |
|------------------|---------------------------------------|
| Koto internal (`/onboard/:id`, etc.) | `I'll pull from Koto.` |

| Malformed URL | `That doesn't look like a URL I can reach.` (amber) |

### 5.1 IngestPanel extension — URL-detection branching (D-29)

**File (extends):** `src/components/kotoiq/launch/IngestPanel.jsx`

4. **Connect GBP button** added in a fourth tab position in the existing tablist. Label: `Connect GBP` with `MapPin` icon. Active tab swaps the input area for the GBP wizard (§5.3).

### 5.5 Source-authenticity visual tier (D-12)

**The problem:** A briefing paragraph containing data from GBP Mode 3 (public) looks almost identical to one from GBP Mode 1 (authenticated) if the citation chip is the only tell. Phase 7 §10 set teal for authenticated sources; Phase 8 introduces a second tier.

**Rule (prescriptive):**
- Authenticated sources → `T` (teal) chip. Confidence halo uses the Phase 7 default (teal at ≥0.85, amber at 0.5–0.85).
- Scraped / public / OCR sources → `AMB` (amber) chip. Confidence halo uses AMB at all confidence levels (never teal, even at 0.85+).

**Why:** Confidence ≠ authenticity. A website scrape can return a high-confidence value for a phone number that IS wrong because it's from a stale cached page; the operator must always SEE that it came from a scrape, not an API. Chip color + halo color combine to convey both dimensions.

**Cross-source discrepancy behavior (inherits Phase 7 §5.6):** When a scraped value and an authenticated value disagree, the Phase 7 discrepancy catcher fires unchanged. The catcher's callout lists each conflicting source with its chip (teal or amber) so the operator picks knowing which is authoritative.

1. **Streaming narration (Phase 7 §5.1)** must still fire when a Phase 8 source lands — e.g. "Reading the uploaded PDF... 4 sections. Pulled 7 fields. Flagging 1 that conflicts with the voice call."
2. **Briefing doc prose (Phase 7 §5.2)** must still read as a single narrative — Phase 8 sources must feed the same prose template, not spawn a separate "external sources" box.
3. **Confidence halos (Phase 7 §5.3)** must still color-code every Phase 8-populated field; the authenticity tier is expressed in the chip, not a new halo color.
4. **Drop zone (Phase 7 §5.4)** already ships the UX — Phase 8 just wires the actual parsers behind it.
5. **Margin notes (Phase 7 §5.5)** should fire new Phase 8-specific observations: "Your website says `response time under 30 minutes`. Add that as a trust anchor?"
6. **Discrepancy catcher (Phase 7 §5.6)** must fire across Phase 8 sources automatically (e.g. GBP says hours X, website says hours Y, onboarding says hours Z → pink callout).
7. **Soft launch gate (Phase 7 §5.7)** should factor in Phase 8 completeness — percentage jumps when an external pull closes gaps.
8. **Clarification cards (Phase 7 §5.8)** can reference Phase 8 sources in their reason text: "Answering unlocks the local-pack schema, and the GBP pull didn't include service area."
9. **In-context hotspots (Phase 7 §5.11)** attach to Phase 8-populated fields unchanged.
10. **Live ribbon (Phase 7 §5.12)** narrates Phase 8 ingests inline: "Pulled Google Business Profile · 12 fields..." — no new ribbon chrome, same 28px teal strip.
