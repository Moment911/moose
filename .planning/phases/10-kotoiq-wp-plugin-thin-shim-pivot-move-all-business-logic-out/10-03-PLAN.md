<objective>
Build the dashboard-side counterpart to the Plan 02 plugin: Ed25519 envelope
signing, wp/v2/* fetch with Application Password Basic auth, end-to-end
pairing flow, and per-site credentials vault (encrypts App Password with
KOTO_AGENCY_INTEGRATIONS_KEK).

This plan delivers the protocol library that Plans 04-09 will call. After
this lands, any dashboard code can run `await shimRpc(site, 'verb.name', args)`
or `await wpFetch(site, '/wp/v2/posts/42')` with full encryption + audit
discipline.

Per CONTEXT.md D-Authentication (USER-LOCKED): Ed25519 short-lived JWT,
60s exp + nonce. shimRpc.ts is where the dashboard side of that contract
lives.

Per D-Keypair-scope (USER-LOCKED): single global keypair — KOTOIQ_SHIM_DASHBOARD_PRIVKEY
is read once per request from process.env, NOT from per-site columns.

Output:
- Three importable client modules (shimRpc, wpFetch, pairSite)
- Vitest test suites for each (mocked transport + mocked Supabase)
- Public index.ts for downstream consumers
</objective>
