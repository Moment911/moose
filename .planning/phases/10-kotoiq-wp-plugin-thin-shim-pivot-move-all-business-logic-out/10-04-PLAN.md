<objective>
Implement the 20 generic core verbs that handle WordPress primitives without
revealing any business logic. These verbs are the building blocks Plans 06-09
will call to port the v3.x modules to dashboard TypeScript.

Per CONTEXT.md decisions:
- Each verb is generic by design — no `seo.*`, no `sitemap.*`, no `redirect.*` (deferred to discretion D-Claude's-Discretion: exact RPC verb names — the planner refined to 20 here, slot 5 covers the remaining 7)
- file.write confined to `wp-content/uploads/kotoiq/**` per research §Risks & Unknowns
- option.update enforces deny-list per research §Risks & Unknowns

Each verb has BOTH a PHP handler (in wp-plugin-kotoiq-shim/includes/rpc/) AND
a TypeScript wrapper (in src/lib/wp-shim/verbs/) so downstream plans have a
typed call site instead of stringly-typed shimRpc invocations.

Output:
- 7 new PHP verb-handler files (one per category)
- verb-table.php updated to point all 20 verbs to real handlers
- 1 TypeScript file with typed wrappers for all 20 verbs
- Vitest test suite covering wrapper input validation + response parsing
</objective>
