    # Generate PEM private + public keys
    openssl genpkey -algorithm ed25519 -outform PEM -out /tmp/kotoiq-shim-priv.pem

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vercel env → dashboard code | KOTOIQ_SHIM_DASHBOARD_PRIVKEY is the most sensitive secret in the project from this plan forward. Anyone with read access to Vercel env can sign valid envelopes for every site in the fleet. |
| User's machine → Vercel | The keypair is generated locally and pasted into Vercel. Anywhere the local PEM file briefly lives is a leak vector. Mitigation: explicit `shred -u` in the checkpoint instructions. |
| Migration SQL file → prod Supabase | The migration must be applied manually (per project rule). The file is committed publicly. No secrets in the SQL itself. |
| Supabase RLS → fleet data | The 4 new tables must enforce agency-only RLS or cross-agency data leak risk. |
